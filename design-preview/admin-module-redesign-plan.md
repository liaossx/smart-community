# 管理端模块改版计划日志

日期: 2026-04-27

## 当前需求

用户要求将管理端页面从简单卡片样式调整为更接近标准后台系统的数据列表页。

本轮优先处理:
- 报修管理

核心目标:
- 不是只改颜色
- 参考成熟后台的数据列表罗列方式
- 页面结构改成“筛选栏 + 状态汇总 + 表头 + 数据行 + 操作列 + 分页”
- 内部模块样式要与当前管理端 UI 风格匹配
- 用户确认报修管理效果后，再逐个迁移其他管理模块

## 本轮设计原则

1. 保留现有接口与业务逻辑。
2. 优先重构信息编排方式，而不是做浅层皮肤替换。
3. 统一按钮、状态标签、筛选器、分页器的后台视觉语言。
4. 支持桌面端优先，窄屏时允许横向滚动表格。
5. 详情、批量操作、分页逻辑继续复用现有行为。

## 报修管理已落地内容

- 页面主体已改为表格型数据页
- 增加顶部概览说明和月份筛选
- 增加状态汇总条并支持点击筛选
- 增加标准查询区域
- 列表主体改为固定表头 + 数据行
- 操作区改为行内按钮
- 保留详情弹窗
- 保留批量处理、批量完成、批量取消、导出

## 后续模块迁移建议

下一批适合按同一结构迁移的模块:
- 工单管理
- 投诉处理
- 访客审核
- 用户管理
- 费用管理

统一迁移标准:
- 顶部统一为查询工具区
- 主体统一为数据表格区
- 状态统一使用胶囊标签
- 行操作统一使用小尺寸按钮
- 分页统一固定在列表底部

## 风险记录

- 当前报修统计数据中的总量来自接口总数，但各状态计数仍可能只基于当前页数据，若后端没有独立统计接口，会存在统计口径偏差。
- 若后续需要完全对齐专业后台表格体验，建议补充:
  - 排序
  - 列显隐
  - 批量跨页选择
  - 独立统计接口

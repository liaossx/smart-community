# 支付失败修复报告 (缺少 userId 参数)

已为您修复支付失败的问题。

## 问题原因
根据您提供的后端代码：
```java
public Result<String> payFee(
    @RequestBody PayFeeDTO dto, 
    @RequestParam("userId") Long userId // 必须的 URL 参数
)
```
后端接口要求必须在 URL 中传递 `userId` 参数（例如 `/api/fee/pay?userId=123`），而不仅仅是在请求体中传递 JSON 数据。
之前的请求只发送了 JSON Body (`{"feeId":4,"payType":"wx","amount":171}`)，缺少了 URL 上的 `userId`，导致后端参数绑定失败 (400 Bad Request)。

## 修复内容
1.  **修改 `owner/pages/payment/pay.vue`**：
    在调用 `request` 时，显式添加 `params` 字段来传递 `userId`。

```javascript
// 修改后
await request({
  url: '/api/fee/pay',
  method: 'POST',
  params: { // 这里会被自动拼接到 URL 后
     userId: userInfo.userId || userInfo.id
  },
  data: { // 这里是 RequestBody
    feeId: parseInt(this.billId),
    payType: this.payMethod,
    amount: Number(this.amount)
  }
})
```

2.  **确认 `utils/request.js`**：
    经检查，您的 `request.js` 已经具备了将 `params` 对象拼接到 URL 后的功能（第 41-52 行），因此 POST 请求也会正确携带 Query 参数。

## 验证方法
请再次尝试支付流程。现在请求 URL 应该会变成类似 `http://.../api/fee/pay?userId=4`，并且携带正确的 JSON Body，支付应该能成功了。

# 支付参数修复报告 (字段名不匹配)

已为您修复支付失败的问题。

## 问题原因
您之前提供的后端 DTO 变更为：
```java
// core-service\src\main\java\com\lsx\core\property\dto\PayFeeDTO.java
public class PayFeeDTO {
    // ...
    private String paymentMethod; // 注意这里是 paymentMethod
    // ...
}
```
但前端发送的请求体中使用的字段名是 `payMethod` (`{"billId":4,"payMethod":"ali","amount":171}`)。
由于字段名不匹配，后端无法正确接收支付方式参数，导致参数校验失败（例如 `paymentMethod` 为 null）从而返回 400 错误。

## 修复内容
在 `owner/pages/payment/pay.vue` 中，修改了发送请求时的参数名：

```javascript
// 修改前
data: {
  billId: parseInt(this.billId),
  payMethod: this.payMethod, // 错误字段名
  amount: Number(this.amount)
}

// 修改后
data: {
  billId: parseInt(this.billId),
  paymentMethod: this.payMethod, // 修正为 paymentMethod
  amount: Number(this.amount)
}
```

## 验证方法
请再次尝试支付流程，现在前后端字段名和类型都已匹配，应该可以支付成功了。
